# 계산기 프로그램 기획서

## 1. 프로젝트 개요

- **목표:** 동아리원이 터미널에서 바로 쓸 수 있는 대화형 계산기
- **대상 사용자:** 파이썬 처음 배우는 동아리 신입 부원
- **핵심 가치:** 사칙연산뿐 아니라 **재무 계산까지 한 번에** 가능해야 함
- **활용 모듈:** 이미 만들어둔 `operations` 패키지의 함수를 그대로 사용 (새로 구현 X)

## 2. 사용자 흐름 (User Flow)

프로그램을 한 번 실행하면, 사용자가 종료를 선택할 때까지 **반복 동작**합니다.

```
[프로그램 시작]
   ↓
1. 카테고리 메뉴 출력 → 사용자 선택
   ↓
2. 세부 연산 메뉴 출력 → 사용자 선택
   ↓
3. 필요한 입력값 받기 (예: 두 숫자, 원금/이율/기간 등)
   ↓
4. 계산 결과 출력
   ↓
5. "계속 / 종료" 묻기 → 계속이면 1번으로
```

## 3. 기능 명세

### 3-1. 카테고리 메뉴

```
========= 계산기 =========
1. 기본 연산
2. 고급 연산
3. 재무 계산
0. 종료
==========================
선택: _
```

### 3-2. 세부 연산 매핑

| 카테고리 | 메뉴 번호 | 함수 | 입력 |
| --- | --- | --- | --- |
| 기본 | 1 | `add` | 숫자 2개 |
| 기본 | 2 | `subtract` | 숫자 2개 |
| 기본 | 3 | `multiply` | 숫자 2개 |
| 기본 | 4 | `divide` | 숫자 2개 |
| 고급 | 1 | `power` | 밑(a), 지수(b) |
| 고급 | 2 | `sqrt` | 숫자 1개 |
| 고급 | 3 | `factorial` | 정수 1개 |
| 재무 | 1 | `simple_interest` | 원금, 이율, 기간(년) |
| 재무 | 2 | `compound_interest` | 원금, 이율, 기간(년) |
| 재무 | 3 | `future_value` | 현재가치, 이율, 기간(년) |
| 재무 | 4 | `present_value` | 미래가치, 이율, 기간(년) |
| 재무 | 5 | `cagr` | 시작값, 종료값, 기간(년) |

## 4. 입력/출력 규격

### 입력
- 숫자 입력은 `float`로 변환 (정수 입력도 허용)
- 이율은 **소수점**으로 받음 (5%면 `0.05`)
- 안내 문구는 **한글**로 표시 (예: `"원금을 입력하세요: "`)

### 출력
- 일반 결과: 소수점 4자리까지만 표시
- **재무 계산 결과:** 금액은 **천 단위 콤마** 적용 (예: `1,628,894.6268`)
- CAGR은 백분율로 변환해서 표시 (예: `14.87%`)

## 5. 에러 처리

다음 상황에서 프로그램이 죽지 않고 **에러 메시지 출력 후 메뉴로 복귀**해야 합니다.

| 상황 | 동작 |
| --- | --- |
| `divide`에서 b=0 | "0으로 나눌 수 없습니다." 출력 후 메뉴 복귀 |
| `sqrt`에 음수 입력 | "음수의 제곱근은 지원하지 않습니다." 출력 |
| `factorial`에 음수/실수 입력 | "0 이상의 정수만 가능합니다." 출력 |
| 숫자 자리에 문자 입력 | "숫자를 입력하세요." 출력 후 다시 입력 받기 |
| 메뉴 번호 잘못 입력 | "올바른 메뉴 번호가 아닙니다." 출력 후 다시 입력 |

## 6. 종료 조건

- 카테고리 메뉴에서 `0` 입력 시 `"계산기를 종료합니다."` 출력하고 프로그램 종료
- 그 외 어떤 화면에서든 `q` 입력 시 즉시 종료


## 9. 구현 시 주의사항

- `calculator.py`는 **`operations` 패키지의 함수를 import해서 사용**할 것 (수식을 직접 다시 작성하지 말 것)
- 함수 분리: 메뉴 출력 / 입력 받기 / 계산 / 결과 포맷팅을 각각 별도 함수로 분리
- `while True` 루프로 반복 동작 구현
- `try / except`로 에러 처리

## 11. HTML 버전 (확장)

CLI 버전(§1~9)과 **동일한 기능**을 브라우저에서도 쓸 수 있도록 확장합니다.

### 11-1. 목표 / 구현 방식

- **목표:** 동아리원이 브라우저에서 바로 쓸 수 있는 시각적 계산기
- **구현 방식:** 순수 프론트엔드 (HTML + CSS + Vanilla JavaScript)
  - 백엔드 서버 없음 → `index.html`을 브라우저로 열기만 하면 동작
  - 계산 로직은 JS로 재구현 (Python `operations` 패키지의 함수 시그니처와 동일한 이름·인자 순서로 맞춤)
- 외부 라이브러리·CDN 사용 ❌ (오프라인에서도 동작해야 함)

### 11-2. 파일 구조

§8에서 정한 `AI_shower_calculator2/` 안에 `web/` 하위로 추가합니다.

```
AI_shower_calculator2/
├── operations/          # CLI용, 그대로 유지 (수정 X)
├── calculator.py        # CLI 진입점
└── web/                 # ← 새로 만들 폴더
    ├── index.html       # 화면 마크업 + 진입점
    ├── style.css        # 스타일
    └── app.js           # UI 제어 + 계산 로직
```

### 11-3. UI 구성

CLI의 "카테고리 → 세부 연산 → 입력 → 결과" 흐름을 한 화면에 옮긴 형태.

```
┌─────────────────────────────────────────┐
│              계산기                      │
├─────────────────────────────────────────┤
│ [기본 연산] [고급 연산] [재무 계산]       │  ← 카테고리 탭 (3개)
├─────────────────────────────────────────┤
│ 세부 연산: [ ▼ 드롭다운 ]                │
│ ─────────────────────────────────────── │
│ 입력1: [           ]                    │
│ 입력2: [           ]                    │  ← 연산에 따라 동적으로
│ 입력3: [           ]                    │     필요한 칸만 표시
│                                         │
│ [        계산하기        ]              │
├─────────────────────────────────────────┤
│ 결과:                                   │
│   >> 복리 이자: 157,625.0000원           │
└─────────────────────────────────────────┘
```

동작 규칙:

- 카테고리 탭 클릭 → 해당 카테고리의 세부 연산 목록으로 드롭다운 갱신
- 세부 연산 선택 → 해당 연산이 요구하는 입력 필드만 보이게 (나머지는 숨김)
- "계산하기" 클릭 → 결과 영역에 결과 한 줄 출력 (이전 결과는 덮어씀)

### 11-4. 기능 매핑

§3-2 표와 **완전히 동일**합니다. JS 함수 이름도 Python과 동일하게 작성:

`add`, `subtract`, `multiply`, `divide`,
`power`, `sqrt`, `factorial`,
`simple_interest`, `compound_interest`, `future_value`, `present_value`, `cagr`.

### 11-5. 입출력 규격

§4를 그대로 따릅니다.

- 입력 필드는 `<input type="number">` 사용, 모든 값은 `parseFloat`로 변환 (이율은 소수, 예: 0.05)
- 안내 문구·라벨은 한글
- 출력 포맷:
  - 일반 결과: 소수점 4자리 (`toFixed(4)`)
  - 재무 결과: 천 단위 콤마 + 소수점 4자리 (예: `1,628,894.6268`) — `Intl.NumberFormat` 사용 권장
  - CAGR: 백분율, 소수점 2자리 (예: `14.87%`)

### 11-6. 에러 처리

§5와 동일한 케이스를 화면에서 처리합니다. 페이지가 새로고침되거나 죽지 않고, 결과 영역에 **빨간 글씨**로 메시지만 표시한 뒤 다시 입력을 바꿔 재계산할 수 있어야 함.

| 상황 | 표시 메시지 |
| --- | --- |
| `divide`에서 b=0 | "0으로 나눌 수 없습니다." |
| `sqrt`에 음수 입력 | "음수의 제곱근은 지원하지 않습니다." |
| `factorial`에 음수/실수 입력 | "0 이상의 정수만 가능합니다." |
| 입력 칸이 비어있거나 숫자가 아님 | "숫자를 입력하세요." |
| 카테고리·세부 연산 미선택 상태에서 계산 시도 | "연산을 선택하세요." |

### 11-7. 그 외 요구사항

- **반응형:** 화면 폭 360px(모바일 세로) ~ 1280px(데스크톱)에서 레이아웃이 깨지지 않을 것
- **접근성 최소선:** 모든 input에 `<label>` 연결, 버튼은 `<button>` 태그 사용
- **종료 개념 없음:** 브라우저 탭을 닫으면 끝 (CLI의 `0`/`q` 종료는 적용되지 않음)
- **상태 저장 없음:** 새로고침하면 초기화 (이력·로컬스토리지 등 사용 X)

### 11-8. 구현 시 주의사항

- `app.js`는 **계산 함수**(operations와 동일 시그니처)와 **UI 제어 함수**를 분리해 작성
- DOM 조작은 `document.getElementById` / `addEventListener` 수준의 바닐라 JS만 사용
- 인라인 스크립트·인라인 스타일 최소화 (가능한 `app.js` / `style.css`에 분리)
