# 실습 따라하기 (명령어 모음)

> **시작 전 공통**: 아래 모든 명령어는 **VS Code 터미널**에 입력합니다.
> 터미널 열기: **`Ctrl` + `` ` ``** (키보드 숫자 `1` 왼쪽에 있는 키, 따옴표처럼 생긴 ` 키)
> 또는 상단 메뉴 **Terminal → New Terminal**

---

## 지금 무슨 일이 일어났는지 (한 줄 요약)

> 나와 팀원이 **둘 다** push 하기 전에 커밋을 만들어서 **원격(GitHub)과 내 컴퓨터의 상태가 두 갈래로 갈라졌다.**
>
> 그래서 GitHub가 "어떻게 합칠지 정해줘"라고 거부한 것. 아래 순서대로 따라 하면 해결됩니다.

---

## 🔵 팀원2 (push 실패한 사람) — 가장 먼저 진행

### 1단계. 터미널에 아래 명령어 입력 후 `Enter`

```bash
git pull --no-rebase
```

### 2단계. 갑자기 검은 화면에 영어가 잔뜩 뜹니다 (당황 금지!)

화면 맨 위 줄에 `Merge branch 'main'...` 같은 글자가 보일 거예요. **수정하지 말고 그냥** 아래 키를 차례로 누르세요.

| 순서 | 키 | 화면 변화 |
|:---:|---|---|
| 1 | `Ctrl + O` | 아래쪽에 `File Name to Write: ...` 라는 줄이 나타남 |
| 2 | `Enter` | 짧게 `[ Wrote 6 lines ]` 같은 메시지가 깜빡함 |
| 3 | `Ctrl + X` | 원래 터미널 화면으로 돌아옴 |

> 💡 `Ctrl + O` = 저장 / `Ctrl + X` = 닫기 라고만 기억하면 됩니다.

### 3단계. 마지막 명령어

```bash
git push
```

→ 에러 없이 끝나면 성공. **완료된 걸 팀원3에게 알려주세요.**

---

## 🟢 팀원3 — 팀원2가 끝났다고 알린 뒤에 진행

### 1단계. 최신 상태 받기

```bash
git pull
```

→ "Already up to date" 또는 파일이 몇 개 받아진다는 메시지가 뜨면 정상.

### 2단계. 자기 이름 파일 만들기

1. VS Code **왼쪽 파일 탐색기** (📁 모양 아이콘)에서 **새 파일 만들기** 아이콘 클릭
   (또는 빈 공간에서 마우스 오른쪽 클릭 → **New File**)
2. 파일 이름은 `자기이름.txt` 예: `홍길동.txt`
3. 파일 안에 아무 내용이나 한 줄 입력 후 **`Ctrl + S`** 로 저장

### 3단계. 터미널에서 아래 3줄 입력 (`자기이름`을 본인 이름으로 바꾸기)

```bash
git add 자기이름.txt
```
```bash
git commit -m "자기이름 파일 추가"
```
```bash
git push
```

→ 에러 없이 끝나면 성공. **완료된 걸 팀원4에게 알려주세요.**

---

## 🟡 팀원4 — 팀원3이 끝났다고 알린 뒤에 진행

### 1단계. 최신 상태 받기

```bash
git pull
```

### 2단계. 자기 이름 파일 만들기

1. VS Code **왼쪽 파일 탐색기**에서 **새 파일** 만들기
2. 파일 이름은 `자기이름.txt`
3. 아무 내용 한 줄 입력 후 **`Ctrl + S`** 로 저장

### 3단계. 터미널에서 아래 3줄 입력 (`자기이름`을 본인 이름으로 바꾸기)

```bash
git add 자기이름.txt
```
```bash
git commit -m "자기이름 파일 추가"
```
```bash
git push
```

→ 완료.

- 화면이 멈춘 것 같다 → 게춘 게 아니라 입력을 기다리는 중일 수 있습니다. 위 안내 순서대로 키를 눌러보세요
